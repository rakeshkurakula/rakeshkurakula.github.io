Building a Real-Time Voice Training System with Whisper.cpp

Rakesh Kurakula • November 2025 • 15 min read

While preparing for distinguished engineer interviews, I realized a gap: my technical depth wasn't matching my communication clarity. I needed a tool that could provide immediate, objective feedback on my speaking—something that could help me reduce my accent, control my pace, and eliminate filler words like "um" and "uh."

Existing solutions like Elsa Speak and Speechling were good for basic pronunciation, but they lacked the customization I needed. I wanted rubric-based scoring (pace, intonation, fillers, pauses), reference audio comparison, and most importantly, near real-time feedback.

This post documents how I built a voice training system using whisper.cpp that delivers feedback in under 150ms—fast enough to feel interactive.

The Problem: Why Existing Tools Weren't Enough

Most voice training apps focus on pronunciation correctness, but they miss the nuances that matter for professional communication:

  • Pace Control: Speaking too fast (180+ WPM) vs optimal (120-150 WPM)
  • Filler Words: "Um", "uh", "like" - confidence killers in interviews
  • Pauses: Awkward silences vs purposeful pauses for emphasis
  • Intonation: Monotone vs engaging pitch variance
  • Clarity: Word pronunciation and articulation

I needed a system that could evaluate all of these criteria simultaneously, provide a composite score, and do it fast enough that I could iterate during practice sessions.

Why Whisper.cpp?

I initially considered several ASR (Automatic Speech Recognition) options:

Options Evaluated:

1. Google Cloud Speech-to-Text
   ✓ Pros: High accuracy (95%+), streaming support
   ✗ Cons: Costs add up ($0.016/min), requires internet, privacy concerns

2. OpenAI Whisper (Python)
   ✓ Pros: SOTA accuracy, free, open-source
   ✗ Cons: 2-3s latency with base model, heavy memory (2GB+)

3. Whisper.cpp
   ✓ Pros: C++ speed, tiny model (72MB), <500ms latency
   ✓ Pros: Runs locally, no API costs, privacy-first
   ✗ Cons: Slightly lower accuracy (89% vs 95%)

Decision: Whisper.cpp's speed advantage was decisive. For pronunciation feedback, 89% accuracy is "good enough"—users can handle occasional misrecognitions. But 2-3 second delays would break the real-time feel.

Whisper.cpp Integration Deep Dive

Whisper.cpp is a C++ port of OpenAI's Whisper model, optimized for inference speed. Here's how I integrated it:

// Build whisper.cpp (one-time setup)
git clone https://github.com/ggerganov/whisper.cpp
cd whisper.cpp
make

// Download Whisper Tiny model (72MB)
bash ./models/download-ggml-model.sh tiny

// Model files
models/
  ├── ggml-tiny.bin          (72MB - fastest)
  ├── ggml-base.bin          (142MB - balanced)
  └── ggml-small.bin         (466MB - accurate)

The Tiny model is surprisingly effective. In my testing across 100 voice samples:

  • Tiny: 89% WER, 450-500ms latency, 72MB
  • Base: 93% WER, 900-1100ms latency, 142MB
  • Small: 95% WER, 2000-2500ms latency, 466MB

For real-time feedback, Tiny was the clear winner. The 4% accuracy drop was imperceptible in practice.
